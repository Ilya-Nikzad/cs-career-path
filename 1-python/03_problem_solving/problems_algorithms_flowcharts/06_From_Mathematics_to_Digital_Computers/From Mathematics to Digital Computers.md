# From Mathematics to Digital Computers

## 1. Algorithms and Programming
- Before writing computer programs, we must first clearly describe algorithms.
- An algorithm is a step-by-step method to solve a problem.
- Computers execute algorithms, but they need a precise, structured input.

## 2. Input in Computers
- Computers have finite memory, so they can only store limited information.
- Integers and rational numbers can be stored exactly.
- Rational numbers are stored as fractions of integers.

## 3. The Problem of Irrational Numbers
- Numbers like π and √2 are irrational (cannot be written as a fraction of integers).
- They cannot be stored exactly in a computer.
- Therefore, they must be approximated.

## 4. Approximation in Computing
- Irrational numbers are replaced by rational approximations.
- Accuracy depends on precision (number of decimal places).
- Higher precision → more accurate results, but uses more memory.

## 5. Example: √2 Problem
- We want to compute √2 numerically.
- Since it is irrational, it cannot be stored exactly.
- We must use an approximation method.

## 6. Heron’s Method (Babylonian Method)
- An ancient iterative algorithm for square roots.

Steps:
- Start with an initial guess (e.g., between 1 and 2 for √2)
- Take the average to get a better approximation
- Repeat the process to improve accuracy

Each iteration gives a more accurate result.

## 7. Key Insight
- Algorithms existed long before computers.
- Computers require finite, representable data.
- Therefore, approximation is essential in digital computation.

## 8. Conclusion
- Inputs and outputs in computing are often approximations.
- Algorithms must respect memory limitations.
- Iterative methods (like Heron’s method) improve precision step by step.
