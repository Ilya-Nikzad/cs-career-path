# Discretisation & Pseudocode Basics

---

## 📘 What is Discretisation?
Discretisation = converting continuous processes into step-by-step actions  

Computers only understand small steps, not smooth change  

---

## 📦 Variables and Assignment in Pseudocode (←)

In pseudocode, we use an arrow (←) for assignment.  
This means: “store this value in this variable”

### Important:
- The variable is created at the same time it is assigned  
- We do NOT need to declare its type (like int, string, etc.)  
- Assignment (←) means “replace old value with new value”  

---

## ⚖️ Comparison Operators in Pseudocode

Comparison operators are used to compare two values and return a result:  
👉 True or False  

### Key points:
- They do NOT change values (unlike assignment)  
- They only ask questions  
- They always return True or False  
- They do NOT change variables  

---

## 🔀 If–Then Logic in Pseudocode

The if–then statement is used to make decisions.  

It means:  
“If a condition is true, do something.”

We often also include:  
“Otherwise, do something else.”

### Key points:
- If–then logic = decision-making in steps  
- It depends on True/False conditions  
- It controls what the algorithm does next  

---

## ❓ Q1: What does discretisation mean in computing?
converting continuous processes into step-by-step (discrete) actions that a computer can follow  

---

## ❓ Q2: What does the assignment arrow ← mean in pseudocode?
store a value into a variable (and replace any previous value)  

---

## 🧠 What you learned:

- Discretisation = turning continuous processes into steps  
- Pseudocode = human-readable algorithm description  
- Assignment (←) = store/overwrite value  
- Comparison (=, <, >) = checks that return True/False  
- If–else = decision making  
- Python translates pseudocode directly into executable logic  

---

## ⚡ What to remember:

👉 Computers think in steps, not smooth changes  
👉 Assignment changes memory  
👉 Comparison only asks questions  
👉 If–else controls decisions
