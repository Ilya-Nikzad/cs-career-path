# Flowchart and Pseudocode

## 1. Conversion Between Flowchart and Pseudocode

A flowchart is a visual diagram that shows steps in a process using shapes and arrows.  
A pseudocode is a text-based way of describing the same process using structured steps.

**Conversion means:**
- Taking a flowchart → writing it as pseudocode  
- Taking pseudocode → drawing it as a flowchart  

Both describe the same logic in different formats.

---

## 2. Breaking a Real-Life Process into Computer Steps

Computers cannot understand “big actions” like brushing teeth directly.  
So we break them into small, controllable steps.

Each step becomes:
- A variable  
- A loop  
- A condition  
- A simple action  

**Example: Brushing Teeth**

Instead of: *Brush teeth*  
We break it into:
- Take toothbrush  
- Apply toothpaste  
- Brush once (repeatable action)  
- Count brushing cycles  
- Stop after required number of brushes  

---

## Q1: What is the main difference between a flowchart and pseudocode?

A flowchart is a visual diagram of a process,  
while pseudocode is a text-based description of the same logic.

---

## Q2: Why do we break real-life actions into smaller steps in programming?

We break real-life actions into smaller steps because computers can only understand simple, precise instructions, not complex “big actions” directly.

---

## Q3: What programming structure is used when we say: “IF awake == True THEN do actions”?

It is an IF statement (conditional statement).

---

## Q4: Why are loops useful in modeling actions like brushing teeth?

Loops are useful because they allow us to repeat an action multiple times automatically until a condition is met.

---

## Final Thinking Model

REAL LIFE → BREAK INTO STEPS → ADD LOOPS + CONDITIONS → PSEUDOCODE  

---

## What to Remember

- Flowchart = visual logic  
- Pseudocode = written logic  
- IF/ELSE = decisions  
- Loops = repetition  
- Variables = states (not actions)  
- Always break big tasks into small steps
