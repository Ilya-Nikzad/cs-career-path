# 📘 Python Collections – Lists vs Dictionaries

A collection is a variable that can store more than one value at the same time.  
That “identification system” is called an index or key (a marker).

So a collection:
- holds multiple items  
- requires a way to locate each item later  
- uses something like a position or tag  

---

## 📋 List

A list stores multiple values using positions.

Key idea:
You retrieve values using a position (index), not a label.

List = access by position

---

## 📚 Dictionary

A dictionary also stores multiple values, but differently.  
Instead of positions, it uses labels (keys).

Key idea:
You retrieve values using a tag (key), not a position.

Dictionary = access by name (key)

---

## ✏️ Updating a Dictionary Item

A dictionary item is updated using its key (label).

Key idea:
You modify values using keys instead of positions.

---

## 🧠 Dictionary Mental Model

A dictionary is like a label-based storage system.

Dictionary = key-based storage ("summer", "fall"...)

---

## ❓ Question 1

What is the key difference between a list and a dictionary in how they access values?

Answer:
A list uses index positions, and a dictionary uses key labels.

---

## 💡 Simple rule:

- If you want to find something by position → use a list  
- If you want to find something by name/label → use a dictionary  

---

## 🧪 Identify the type

Situation:
You want to store:
- "Ali" → 90  
- "Sara" → 85  
- "John" → 70  

Answer:
Dictionary

Because it uses names (keys), not positions.
